# 代码分析 Skill

搜索和阅读服务器代码、客户端代码，理解业务逻辑。用于排查问题时定位相关代码逻辑。

## 何时使用

当你需要理解某个功能的实现逻辑时使用，包括但不限于：
- 查看某个接口的服务器处理逻辑
- 查看客户端某个功能的实现方式
- 理解配置是如何被加载和使用的
- 根据错误码找到对应的错误处理代码
- 根据日志中的关键词定位到具体代码位置

## 代码路径

所有仓库路径从 `resolve_env.py` 输出中获取（如 `server_path`、`client_path`、`config_path`、`gm_path` 等），无需直接读取 config.yaml 的 code 段。

## 使用方法

代码分析不需要专门的 Python 工具，直接使用你自带的文件搜索和读取能力。

### 1. 搜索代码

使用 Grep 工具在代码目录中搜索关键词：

- 搜索业务关键词（如功能名称、接口名、错误码）
- 搜索类名、方法名、配置引用

### 2. 阅读代码

使用 Read 工具读取找到的文件，理解业务逻辑。

### 3. 常见搜索策略

**根据接口名/命令号定位代码：**
1. 从日志中获取接口名或命令号
2. 在服务器代码中搜索该标识
3. 找到对应的处理函数，阅读逻辑

**根据错误码定位代码：**
1. 从日志或 GM 数据中获取错误码
2. 在代码中搜索该错误码的定义
3. 反向找到产生这个错误码的代码位置

**根据业务名定位配表（见 `knowledge/SKILL.md`）：**
1. 在服务器代码中搜索业务名相关关键词
2. 找到配表类/引用 → 获取配表文件名
3. 用 excel_reader 工具读取配表

## 注意事项

- 具体的代码组织方式、模块结构、配表引用模式等因项目而异，详见 `knowledge/SKILL.md` 中的项目业务知识
- 代码搜索时先用精确关键词，搜不到再用模糊关键词
- 阅读代码时注意看函数调用链，不要只看单个函数
