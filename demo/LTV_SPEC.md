# LTV Excel 报告生成 —— 完整规格文件

> 版本: v2.0 (2026-04-09)
> 用途: 从原始付费日志 (input.xlsx) 生成 LTV 分析报告 (output.xlsx)

---

## 1. 输入文件格式

### 1.1 文件结构
| 项目 | 值 |
|------|-----|
| Sheet 名 | `GA-SQL工具` |
| 典型行数 | ~9,200 行 |
| 列数 | 38 列 |

### 1.2 列定义

| 列名 | 类型 | 说明 |
|------|------|------|
| `day` | 日期 | 数据日期, 如 `2024-05-01` |
| `adtrace_platform` | 字符串 | 渠道标识, 如 `Channel_A`, `Channel_B`, `Channel_C` |
| `reattr_seq` | 整数 | 归因序号: 0=新用户, ≥1=老用户 |
| `device_os` | 字符串 | 设备系统: `IOS`, `安卓`, `其他` |
| `new_user` | 整数 | 新增用户数 |
| `pay1` ~ `pay15` | 数值 | 第1天到第15天的累计付费 (逐日) |
| `pay30`, `pay45`, `pay60`, `pay90` | 数值 | 第30/45/60/90天的累计付费 |
| `pay120` ~ `pay270` | 数值 | 更长周期的累计付费 (每30天一档) |
| `pay300` ~ `pay720` | 数值 | 超长周期累计付费 (**本工具不使用**) |
| `total_pay` | 数值 | 总付费 (**本工具不使用**) |

### 1.3 实际使用的付费列 (仅14列)

```
pay1, pay3, pay7, pay15, pay30, pay45, pay60, pay90,
pay120, pay150, pay180, pay210, pay240, pay270
```

> ⚠️ `pay2`, `pay4`~`pay6`, `pay8`~`pay14`, `pay300`+ 均**不使用**。

## 2. 核心计算规则

### 2.1 净付费系数

所有付费列在聚合前，必须先乘以 `device_os` 对应的净付费系数：

| device_os | 系数 | 说明 |
|-----------|------|------|
| `IOS` | × 0.8415 | 苹果抽成后净收入 |
| `安卓` | × 0.51 | 安卓渠道抽成后净收入 |
| `其他` | × 0.8075 | 其他平台净收入 |

```python
NET_COEFFICIENTS = {'IOS': 0.8415, '安卓': 0.51, '其他': 0.8075}
```

- `new_user` **不乘系数**，保持原值
- 净付费 = 原始付费 × 系数
- 聚合后取整 (`round()`)

### 2.2 数据分组

每条数据按三个维度分组：

| 维度 | 规则 |
|------|------|
| **月份** | 从 `day` 提取年月 (如 `2024-05`) |
| **渠道** | `adtrace_platform` 值 |
| **用户分段** | `整体`: 全部行; `新用户`: `reattr_seq == 0`; `老用户`: `reattr_seq != 0` |

### 2.3 不完整月份裁剪

如果某月的实际有数据天数 < 该月日历天数的一半，则排除该月。

> 例: 2025-03 只有 9 天数据，日历天数 31 天，9 < 15.5，排除。

### 2.4 三角结构

较新月份的观测窗口较短，超出观测期的高天数列为空。

判断方式：
- 计算每月月末到数据最后日期的天数差 = 观测天数
- 如果某 pay 天数 > 观测天数，该列为空 (None)
- 同时，如果某列的原始聚合值全为 0 或 NaN，也视为空

### 2.5 聚合公式

对每个 (月份, 渠道, 用户分段) 组合：

```
新增数 = SUM(new_user)                    # 不乘系数
首日付费 = ROUND(SUM(pay1 × 系数))         # 乘系数后求和取整
3日付费 = ROUND(SUM(pay3 × 系数))
...
270日付费 = ROUND(SUM(pay270 × 系数))
```

## 3. 输出文件结构

### 3.1 Sheet 列表 (10 个)

```

### 3.2 Detail Sheet 布局 (每个渠道×分段)

列 Q~R: 预留 (用于 300日/330日/360日 的增长率和倍率链)
```

#### 列名映射 (14个付费档位)

| ... | ... | ... | ... | ... |
| pay270 | 270日付费 | 270日LTV | 270日 | 300日 |
| — | — | — | — | 330日 (扩展) |
| — | — | — | — | 360日 (扩展) |

### 3.3 Section 1: 分月付费

| 行号 | 内容 |
|------|------|
| R1 | `['分月付费', None, None, ...]` (18列) |
| R2 | `['月', '新增数', '首日付费', '3日付费', '7日付费', '15日付费', '30日付费', '45日付费', '60日付费', '90日付费', '120日付费', '150日付费', '180日付费', '210日付费', '240日付费', '270日付费', None, None]` |
| R3 ~ R(2+N) | 每月一行数据 |

数据行格式：
```
A列: datetime(year, month, 1)    # 该月第一天
B列: SUM(new_user)               # 整数, 不乘系数
C列: ROUND(SUM(净pay1))          # 整数, 已乘系数
D列: ROUND(SUM(净pay3))
...
P列: ROUND(SUM(净pay270))        # 或 None (三角结构)
Q~R列: None
```

### 3.4 Section 2: 分月LTV

| 行号 | 内容 |
|------|------|
| R(N+3) | 空行 |
| R(N+4) | `['分月LTV', None, ...]` |
| R(N+5) | `['月', '首日LTV', '3日LTV', '7日LTV', '15日LTV', '30日LTV', '45日LTV', '60日LTV', '90日LTV', '120日LTV', '150日LTV', '180日LTV', '210日LTV', '240日LTV', '270日LTV', None, None, None]` |
| R(N+6) ~ R(2N+5) | 每月一行 Excel 公式 |

**公式规则**：
```
首日LTV = =C{pay_row}/$B{pay_row}     # C列(首日付费) / B列(新增数)
3日LTV  = =D{pay_row}/$B{pay_row}     # D列(3日付费) / B列(新增数)
...
```

其中 `pay_row` 是 Section 1 中对应月份的行号 (1-based, Excel 行号)。

> 注意: B列用 `$B` (绝对列引用), C~P列用相对引用。

> 三角空位: 如果 Section 1 该位置为空, 则 LTV 也为空 (None, 不写公式)。

### 3.5 Section 3: 分月倍率

| 行号 | 内容 |
|------|------|
| R(2N+6) | 空行 |
| R(2N+7) | `['分月倍率', None, ...]` |
| R(2N+8) | `['月', '3日', '7日', '15日', '30日', '45日', '60日', '90日', '120日', '150日', '180日', '210日', '240日', '270日', None, None, None, None]` |
| R(2N+9) ~ R(3N+8) | 每月一行 Excel 公式 |

**公式规则**：
```
倍率 = LTV_X / LTV_首日

3日倍率 = =C{ltv_row}/$B{ltv_row}     # 3日LTV / 首日LTV
7日倍率 = =D{ltv_row}/$B{ltv_row}     # 7日LTV / 首日LTV
...
```

> ⚠️ 注意: 倍率没有"首日"列 (首日/首日=1 无意义), 从"3日"开始。

### 3.6 Section 4: 分月增长率

| 行号 | 内容 |
|------|------|
| R(3N+9) | 空行 |
| R(3N+10) | 空行 |
| R(3N+11) | `['分月增长率', None, ...]` |
| R(3N+12) | `['月', '3日', '7日', '15日', '30日', '45日', '60日', '90日', '120日', '150日', '180日', '210日', '240日', '270日', '300日', '330日', '360日', None]` |
| R(3N+13) ~ R(4N+12) | 每月一行 Excel 公式 |
| R(4N+13) | 平均值行 |

> 比前面的 Section 多了 300日/330日/360日 三列。

**公式规则**：
```
增长率 = LTV_X / LTV_{X-1}  (当前档位LTV / 前一档位LTV)

3日增长率   = =C{ltv_row}/B{ltv_row}     # 3日LTV / 首日LTV
7日增长率   = =D{ltv_row}/C{ltv_row}     # 7日LTV / 3日LTV
15日增长率  = =E{ltv_row}/D{ltv_row}     # 15日LTV / 7日LTV
30日增长率  = =F{ltv_row}/E{ltv_row}     # 30日LTV / 15日LTV
...
270日增长率 = =N{ltv_row}/M{ltv_row}     # 270日LTV / 240日LTV
```

> ⚠️ 增长率公式引用的是 **Section 2 (LTV)** 的行号, 不是 Section 1 (付费)。
> ⚠️ 300日/330日/360日 在所有月度数据行中均为空 (None)。

**平均值行**：
```
A列: '平均值'
B列(3日):  =AVERAGE(B{first_gr_row}:B{last_gr_row})
C列(7日):  =AVERAGE(C{first_gr_row}:C{last_valid_row_for_this_col})
...
```

- AVERAGE 范围是**动态**的: 每列只包含有数据的月份行
- 300日/330日/360日 的平均值 = **硬编码常数 1**

### 3.7 Section 5: Bottom 汇总

| 行号 | 内容 |
|------|------|
| R(4N+14) | 空行 |
| R(4N+15) | `[sheet名, None, ...]` 如 `['Channel_A-整体']` |
| R(4N+16) | `['倍率', '首日', '3日', '7日', '15日', '30日', '45日', '60日', '90日', '120日', '150日', '180日', '210日', '240日', '270日', '300日', '330日', '360日']` |
| R(4N+17) | 倍率链数据行 |
| R(4N+18) | 空行 |
| R(4N+19) | `['模型', '首日', '3日', ...]` |
| R(4N+20) | 模型数据行 1 (整体 seed) |
| R(4N+21) | 模型数据行 2 (新用户 seed) |
| R(4N+22) | 模型数据行 3 (老用户 seed) |

#### 倍率链

倍率链通过链式乘积，将平均增长率累积为从首日到各天数的综合倍率：

```
B列(首日) = 1                              # 基准值
C列(3日)  = =B{this}*B{avg}               # 前一列 × 对应平均增长率
D列(7日)  = =C{this}*C{avg}               # 前一列 × 对应平均增长率
...
R列(360日) = =Q{this}*Q{avg}
```

其中 `{avg}` 是 Section 4 平均值行的行号。

> 逻辑含义: 倍率链[X日] = ∏(平均增长率[3日] × 平均增长率[7日] × ... × 平均增长率[X日])

#### 模型行

用 seed (种子值/首日LTV目标) × 倍率链, 预测各天数的 LTV：

```
A列: None
B列(首日) = seed 值 (如 0.0085)
C列(3日)  = =$B{this}*C{ratio_row}        # seed × 倍率链[3日]
D列(7日)  = =$B{this}*D{ratio_row}        # seed × 倍率链[7日]
...
```

> B列用 `$B` 绝对列引用, 其余列相对引用。

#### 默认 Seed 值

```python
DEFAULT_SEEDS = {
    'Channel_A': [0.0085, 0.0075, 0.0065],  # [整体, 新用户, 老用户]
    'Channel_B': [0.011,  0.01,   0.009 ],
    'Channel_C': [0.02,   0.018,  0.016 ],
}
```

> 每个 Detail Sheet 的 Section 5 都放**3行模型行** (整体/新用户/老用户的 seed)。

## 6. 关键实现细节

### 6.1 AVERAGE 范围的动态计算

Section 4 平均值行中, 每列的 AVERAGE 范围不同:

```python
# 对于 Section 4 中的第 j 列 (0-based, j=0 对应 3日增长率):
# 该列有数据的月份数 = 有多少个月份在三角结构中该位置非空
# AVERAGE 范围 = 从第一个数据月份的增长率行 到 最后一个有数据月份的增长率行

# 例 (N=10):
# 3日增长率: 所有10个月都有 → AVERAGE(B43:B52)
# 7日增长率: 9个月有 (最新月份没有7日数据) → AVERAGE(C43:C51)
# 270日增长率: 只有1个月有 → AVERAGE(N43)  (实际是N43:N43, 或直接 =N43)
```

### 6.2 增长率公式的列偏移

增长率的"当前"和"前一档"引用的是 **Section 2 (LTV)** 的列:

```
增长率表头:  月  | 3日  | 7日  | 15日 | 30日  | ...  | 270日 | 300日 | 330日 | 360日
增长率公式:     | C/B  | D/C  | E/D  | F/E   | ...  | N/M   | (空)  | (空)  | (空)
               ↑ LTV行的列字母
```

### 6.3 倍率链的列对齐

```
倍率链表头: 首日 | 3日     | 7日     | 15日    | ... | 360日
倍率链公式: =1  | =B*B_avg| =C*C_avg| =D*D_avg| ... | =Q*Q_avg
平均值行:       | B_avg   | C_avg   | D_avg   | ... | Q_avg=1
               ↑ 倍率链和平均值行使用相同的列字母
```

## 7. 命令行接口

## 8. 已知约束与注意事项

1. **数据源微差**: 直接聚合 input 与 result 之间存在 ±1~±5 的整数级微差 (不同版本数据导致), 属正常现象
2. **线性外推数据**: input 中 pay120 以后的某些行存在线性外推 (30天增量恒定), 这是上游数据的特征, 不影响工具逻辑
3. **渠道过滤**: 默认只取行数最多的前 3 个渠道, 小渠道 (如 Channel_D/E) 自动忽略
4. **Excel 公式 vs 计算值**: Section 2/3/4/5 在 Detail Sheet 中全部使用 **Excel 公式** (非硬编码计算值), 便于用户在 Excel 中查看和调试
5. **汇总 Sheet**: 使用 Python 计算值填入 (非跨 Sheet 公式), 因为 openpyxl 不支持跨 Sheet 公式引用
6. **三角结构的判断**: 基于"月末到数据最后日期的天数差", 而非简单的列是否有值; 同时辅以实际聚合值是否为 0 的检查
